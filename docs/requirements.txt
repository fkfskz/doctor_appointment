================================================================
ТРЕБОВАНИЯ К ИНТЕГРАЦИИ МОДУЛЯ
Проект: Сервис записи на прием к врачу
Дата: 05.03.2026
================================================================

1. ЦЕЛЬ ПРОЕКТА
   Разработать веб-сервис для самостоятельной записи пациентов
   на прием к врачам городской поликлиники №1.

2. ФУНКЦИОНАЛЬНЫЕ ТРЕБОВАНИЯ К МОДУЛЮ

   2.1. Модуль должен предоставлять следующие функции:
        - get_doctors() - получение списка врачей
        - get_available_slots(doctor_id, date) - получение свободного времени
        - check_availability(doctor_id, date, time) - проверка доступности
        - make_appointment(data) - создание записи
        - get_appointments_by_phone(phone) - поиск записей по телефону
        - cancel_appointment(id) - отмена записи

   2.2. Форматы данных:
        - Входные: doctor_id (int), date (str: ГГГГ-ММ-ДД), 
          time (str: ЧЧ:ММ), patient_name (str), patient_phone (str)
        - Выходные: dict с ключами success, ticket_number, error

3. НЕФУНКЦИОНАЛЬНЫЕ ТРЕБОВАНИЯ

   3.1. Модуль должен быть написан на Python 3.8+
   3.2. Данные хранятся в JSON файле (не требуется БД)
   3.3. Модуль не должен зависеть от внешних сервисов
   3.4. Код должен соответствовать PEP 8

4. ТРЕБОВАНИЯ К ИНТЕГРАЦИИ

   4.1. Модуль интегрируется в Flask-приложение
   4.2. Основное приложение вызывает функции модуля
   4.3. Модуль не должен содержать код веб-интерфейса

5. КРИТЕРИИ ПРИЕМКИ
   5.1. Пациент может записаться к врачу
   5.2. Пациент не может записаться на занятое время
   5.3. Пациент может проверить свои записи по телефону
   5.4. Пациент может отменить запись
